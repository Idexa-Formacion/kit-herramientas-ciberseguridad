# Checklist — Controles ISO/IEC 27001:2022

> Mapeo rápido y accionable de los controles del Anexo A de la norma. Esta lista no sustituye una auditoría formal, pero sirve como guía operativa para revisar el estado de un SGSI.
>
> Para una formación oficial sobre la norma, consulta el curso [ISO/IEC 27001:2022 Foundation](https://idexaformacion.com/iso-iec-27001-foundation/) de Idexa Formación.

---

## A.5 — Controles organizativos

- [ ] **5.1** Existe una Política de Seguridad de la Información aprobada por dirección.
- [ ] **5.2** Roles y responsabilidades de seguridad están definidos y comunicados.
- [ ] **5.3** Hay segregación de funciones para tareas críticas.
- [ ] **5.7** Existe gestión documentada de inteligencia de amenazas.
- [ ] **5.9** Inventario actualizado de activos de información.
- [ ] **5.10** Política de uso aceptable de los activos.
- [ ] **5.15** Política de control de acceso documentada.
- [ ] **5.23** Seguridad en el uso de servicios cloud.
- [ ] **5.24** Plan de gestión de incidentes de seguridad.
- [ ] **5.30** Continuidad de TIC ante interrupciones.
- [ ] **5.34** Privacidad y protección de datos personales (RGPD).

## A.6 — Controles de personas

- [ ] **6.1** Verificaciones previas a la contratación.
- [ ] **6.2** Términos y condiciones de empleo incluyen seguridad.
- [ ] **6.3** Programa de concienciación, educación y formación en seguridad.
- [ ] **6.4** Proceso disciplinario por incumplimientos.
- [ ] **6.7** Política de teletrabajo segura.
- [ ] **6.8** Mecanismo para reportar eventos de seguridad.

## A.7 — Controles físicos

- [ ] **7.1** Perímetros de seguridad física definidos.
- [ ] **7.2** Controles de entrada físicos en zonas seguras.
- [ ] **7.4** Vigilancia de la seguridad física.
- [ ] **7.7** Política de mesas y pantallas limpias.
- [ ] **7.10** Gestión de soportes de almacenamiento (cifrado, destrucción).

## A.8 — Controles tecnológicos

- [ ] **8.1** Dispositivos de usuario configurados y gestionados.
- [ ] **8.2** Privilegios de acceso restringidos al mínimo necesario.
- [ ] **8.3** Restricción del acceso a la información.
- [ ] **8.5** Autenticación segura (MFA donde aplique).
- [ ] **8.7** Protección contra malware en endpoints y servidores.
- [ ] **8.8** Gestión de vulnerabilidades técnicas (parcheo).
- [ ] **8.9** Gestión de configuraciones (baselines).
- [ ] **8.10** Borrado seguro de información.
- [ ] **8.11** Anonimización/seudonimización de datos cuando sea necesario.
- [ ] **8.12** Prevención de fugas de datos (DLP).
- [ ] **8.13** Copias de seguridad de información (probadas).
- [ ] **8.15** Logs de eventos generados, conservados y revisados.
- [ ] **8.16** Monitorización continua de actividades.
- [ ] **8.20** Seguridad de redes (segmentación, controles).
- [ ] **8.23** Filtrado web.
- [ ] **8.24** Uso de criptografía conforme a una política.
- [ ] **8.28** Codificación segura en desarrollo.
- [ ] **8.32** Gestión de cambios.

---

## Cómo usar este checklist

1. **Recorre cada control** y marca su estado: cumplido ✅, parcial ⚠️, no aplica ➖, o no cumplido ❌.
2. **Para cada control no cumplido o parcial**, abre una acción correctiva con responsable y plazo.
3. **Repite la revisión** de forma periódica (mínimo anual o tras cambios significativos).
